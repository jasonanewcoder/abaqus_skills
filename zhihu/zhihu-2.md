# 拒绝“智障AI”：三步打造你的Abaqus脚本助手

## 引言：为什么你的AI写不好Abaqus脚本？

在上一篇文章中，我们讨论了AI辅助CAE仿真的潜力和挑战。有读者留言说：“道理我都懂，但具体怎么做？”

确实，这是一个非常实际的问题。

很多工程师兴冲冲地找AI帮忙写Abaqus脚本，结果得到一堆无法运行的代码——API调用错误、参数张冠李戴、缺少关键步骤。久而久之，大家对AI失去了信心，宁可自己手动操作或者翻阅厚重的官方文档。

问题出在哪里？

**答案是：AI不知道“Abaqus应该怎么写”，它只知道“Abaqus大概长这样”。**

就像一个只见过几次中文的外国人，你让他写一篇技术论文，他只能硬着头皮用有限的词汇拼凑，结果自然是惨不忍睹。

解决方案也很简单：**给它一本“专业词典”**。

今天，我就手把手教你如何利用Abaqus Skills技能库，让AI真正成为你的Abaqus脚本助手。

---

## 一、核心思路：把AI变成“查字典”的人

### 1.1 传统方式的困境

在没有技能库的情况下，你与AI的对话可能是这样的：

> **你**：帮我写一个Abaqus悬臂梁分析脚本
> 
> **AI**：（自信地写了一段代码）
> 
> **你**：运行报错，说`part.BaseSolidExtrude`不存在
> 
> **AI**：抱歉，我搞错了，应该是`part.BaseSolidExtrude(sketch=sketch, depth=length)`……（其实还是错的）

这种“改错-再错-再改”的循环，不仅效率低下，而且严重打击使用AI的积极性。

### 1.2 正确的方式：引导AI查“字典”

正确的使用方式，是把技能库当作AI的“参考书”：

> **你**：我要做一个悬臂梁分析，请参考skills技能库中的建模、材料、网格、静力分析相关技能来生成代码
> 
> **AI**：好的，我先查一下技能库中的相关模块……根据skill_modeling.md中的长方体模板和skill_material.md中的弹性材料模板，我为你生成了以下代码……

这就是技能库的核心价值：**不是让AI凭记忆写代码，而是让它按图索骥查资料**。

## 二、实战操作：三步配置你的AI助手

### 2.1 步骤一：获取技能库

首先，你需要获取技能库的内容。有两种方式：

**方式A：克隆完整项目（推荐）**

```bash
# GitHub
git clone https://github.com/jasonanewcoder/abaqus_skills.git

# 或 Gitee（国内访问更快）
git clone https://gitee.com/jasonfun1995/abaqus_skills.git
```

**方式B：只复制需要的技能文件**

如果你只需要部分技能，可以直接复制对应文件到本地。比如你需要建模和材料相关技能：

```
├── abaqus_skills/
│   ├── abaqus_scriptcontrol_skills/
│   │   ├── general/skill_modeling.md    # 建模
│   │   ├── general/skill_material.md    # 材料
│   │   ├── general/skill_bc_load.md    # 边界/载荷
│   │   └── ...
```

### 2.2 步骤二：配置AI提示词

这是最关键的一步。你需要告诉AI：“你需要先查技能库，再写代码。”

我建议使用以下提示词模板（可以直接复制使用）：

```
我需要你帮我生成Abaqus Python脚本。请按照以下流程操作：

1. 首先，理解我的需求：[描述你的分析需求]

2. 然后，从以下技能文件中查找相关模板：
   - 建模相关：skill_modeling.md
   - 材料相关：skill_material.md
   - 网格相关：skill_mesh.md
   - 边界/载荷相关：skill_bc_load.md
   - 分析步相关：skill_step.md
   - 作业提交相关：skill_job.md

3. 根据查到的模板生成代码，确保：
   - API调用与技能库中一致
   - 参数设置遵循最佳实践
   - 代码包含中文注释

4. 生成代码后，说明：
   - 使用的技能模块
   - 关键参数的意义
   - 运行方法

请开始吧。
```

### 2.3 步骤三：运行和验证

生成的代码需要验证才能使用。推荐的工作流程：

```
1. 将代码保存为 .py 文件（如 cantilever_beam.py）

2. 在 Abaqus CAE 中运行：
   - File → Run Script → 选择 cantilever_beam.py

3. 检查结果：
   - 模型树是否正确创建
   - 网格是否成功划分
   - 作业是否成功提交
   - 结果是否合理
```

## 三、高级技巧：没有环境也能让AI帮忙

### 3.1 不想装Abaqus？没关系

很多读者问：“我电脑上没装Abaqus，能不能也让AI帮我写脚本？”

答案是：**当然可以**。

核心思路是：**把技能库的内容复制给AI，让它“就地查询”**。

具体操作：

1. **复制技能文件内容**：打开技能库中的.md文件，复制代码模板部分
2. **发送给AI**：告诉AI“这就是Abaqus的API标准写法，请参考它来写代码”
3. **AI生成代码**：由于有了准确的参考，AI生成的代码质量会大幅提升

### 3.2 一个“偷懒”的技巧

如果你连复制都懒得做，我再教你一个更简便的方法：

**直接让AI学习“AI指南”文件**。

项目中有专门的AI使用指南文件（`prompts/ai_guide.md`），它详细说明了：

- 如何正确向AI描述需求
- 哪些问题是必须澄清的
- 常见的错误如何避免

你可以直接把这份指南发给AI：

```
请先阅读以下AI使用指南，理解Abaqus脚本生成的正确流程，然后再帮我写代码。
[复制 ai_guide.md 的内容]
```

## 四、避坑指南：这些错误不要犯

### 4.1 错误一：贪多求全

有些用户一次性把所有技能文件都发给AI，以为“给的信息越多越好”。

**错！**

信息过载会导致AI“选择困难”，不知道该用哪个模板。正确做法是：**只发送与当前需求相关的技能文件**。

比如你要做一个静力分析，就发：
- skill_modeling.md
- skill_material.md
- skill_static_linear.md（静力分析专用）

不要发疲劳、热应力相关的技能文件。

### 4.2 错误二：不做需求澄清

AI不是肚子里的蛔虫，它需要你明确需求才能写对代码。

**正确做法**：在发起请求时，明确说明：
- 分析类型（静力/动力/热力）
- 几何特征（尺寸、形状）
- 材料参数（弹性模量、泊松比）
- 载荷条件（大小、方向、类型）
- 边界条件（固定/简支/自由）

### 4.3 错误三：不验证就使用

这是最严重的错误。

AI生成的代码可能存在各种问题：
- API调用错误（虽然少了）
- 参数设置不合理
- 逻辑遗漏（如忘记划分网格）

**一定要验证**：运行代码、检查模型、对比结果。
