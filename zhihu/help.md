# 不需要翻墙也能用：GitHub Skills 的本地化使用指南

## 前言：被挡在门外的尴尬

很多读者看完我的专栏文章后，兴奋地去尝试配置AI辅助Abaqus脚本开发的环境，结果却遇到了一个尴尬的问题：**Claude Code 在中国大陆几乎无法正常使用**。

网络连接不稳定、API调用频繁失败、安装包下载中断……这些问题让很多读者望而却步。有人甚至留言说：“看教程看了一半，发现连门都进不去，太难了！”

其实，我想说的是：**技能库的价值不在于载体，而在于内容本身。** 即使没有Claude Code，你依然可以充分利用GitHub上的技能库资源。今天这篇文章，我就手把手教你如何在没有Claude Code的情况下，依然高效地使用这些技能。

---

## 一、问题分析：为什么 Claude Code 在中国大陆寸步难行？

### 1.1 基础设施建设层面的困境

Claude Code 依赖 Anthropic 的服务器进行通信，而 Anthropic 的服务在中国大陆没有设立数据中心。这意味着每次调用都需要跨越防火墙，导致：

- **连接不稳定**：高峰期经常超时或断开
- **响应延迟高**：正常情况下几百毫秒的响应时间，在网络不稳定时可能变成几秒甚至几十秒
- **API调用失败**：频繁出现 "Connection Reset"、"Timeout" 等错误

### 1.2 安装层面的障碍

除了网络问题，Claude Code 的安装本身也是困难重重：

- **下载速度慢**：官方安装包体积较大，国内下载经常中断
- **依赖环境缺失**：需要 Node.js、Python 等环境，配置过程繁琐
- **版本更新频繁**：每次更新都需要重新下载安装

### 1.3 真正的核心问题

但我想指出一个更重要的问题：**Claude Code 只是一个载体，它的核心价值在于技能库的内容**。技能库本质上就是结构化的知识文档，包含代码模板、最佳实践、常见错误等内容。

**也就是说，即使没有 Claude Code，你依然可以直接使用这些技能文档。**

---

## 二、解决方案：脱离 Claude Code 的四种替代方案

### 方案一：直接阅读 + 手动调用（最简单）

**核心思路**：把技能库当作“技术手册”，自己查阅文档，编写提示词发给其他AI工具。

**操作步骤**：

1. **克隆技能库到本地**
   ```bash
   # GitHub（推荐）
   git clone https://github.com/jasonanewcoder/abaqus_skills.git
   
   # 或 Gitee（国内访问更快）
   git clone https://gitee.com/jasonfun1995/abaqus_skills.git
   ```

2. **打开技能文件**：根据你的需求，选择对应的技能文档。例如要做静力分析，就打开 `skill_static_linear.md`

3. **提取关键信息**：阅读文档中的代码模板、参数说明、最佳实践

4. **使用其他AI工具**：将技能库中的内容复制给你的AI助手（如ChatGPT、文心一言、通义千问等），让它参考生成代码

**优点**：无需任何特殊配置，完全本地化
**缺点**：需要手动查找和复制内容

### 方案二：本地知识库 + 提示词工程（进阶）

**核心思路**：将技能库导入到本地知识库工具中，实现快速检索和智能问答。

**推荐工具**：

| 工具名称 | 特点 | 适用场景 |
|---------|------|----------|
| **Anything LLM** | 开源免费，支持多种文档格式 | 个人使用 |
| **LangChain-ChatGLM** | 本地部署，无需联网 | 技术能力较强 |
| **ChatPDF** | 快速构建文档问答 | 快速上手 |

**操作步骤（以 Anything LLM 为例）**：

1. 下载并安装 Anything LLM
2. 创建一个新的 Workspace
3. 将 `abaqus_skills` 目录下的所有 `.md` 文件拖入
4. 开始提问："我想做一个悬臂梁静力分析，帮我生成代码"

**优点**：交互友好，类似 Claude Code 的体验
**缺点**：需要一定的配置时间

### 方案三：浏览器插件 + 在线AI（轻量级）

**核心思路**：使用浏览器插件增强在线AI的能力，绕过网络限制。

**推荐组合**：

- **Kimi浏览器插件** + **Kimi AI**：国产工具，网络稳定
- **沉浸式翻译插件** + **ChatGPT**：需要科学上网
- **微信读书插件** + **文心一言**：完全本地化

**具体操作**：

1. 安装 Kimi 浏览器插件
2. 打开技能库页面，点击插件图标
3. 输入需求："请参考当前页面的代码模板，帮我写一个xxx脚本"

**优点**：安装简单，门槛低
**缺点**：依赖浏览器和网络

### 方案四：VS Code + 插件生态（开发者向）

**核心思路**：利用 VS Code 的插件生态，构建类似的智能辅助功能。

**推荐插件**：

| 插件名称 | 功能 |
|---------|------|
| **GitHub Copilot** | 代码自动补全（需要科学上网） |
| **Codeium** | 免费代码补全替代品 |
| **ChatGPT - VSCode** | 在 VS Code 中直接调用 AI |
| **Abaqus Python** | Abaqus 语法高亮和代码片段 |

**配置步骤**：

1. 安装 VS Code 和上述插件
2. 将技能库文件添加到 VS Code 工作区
3. 使用插件的 "Ask AI" 功能，引用技能库内容生成代码

**优点**：与开发环境深度集成
**缺点**：部分插件需要特殊网络环境

## 三、实战案例：不使用 Claude Code 的完整工作流

让我用一个具体例子，展示如何在没有 Claude Code 的情况下完成一次完整的脚本开发。

### 案例需求

生成一个Abaqus脚本，实现：**三维实体悬臂梁的静力分析**

### 步骤一：确定需要的技能文件

根据需求，查看目录结构：
- `general/skill_modeling.md` — 建模基础
- `general/skill_material.md` — 材料定义
- `general/skill_mesh.md` — 网格划分
- `general/skill_bc_load.md` — 边界条件和载荷
- `general/skill_step.md` — 分析步设置
- `general/skill_job.md` — 作业提交
- `static/skill_static_linear.md` — 静力分析

### 步骤二：阅读技能文档

打开 `skill_static_linear.md`，找到代码模板：

```python
# 静力分析模板（简化版）
from abaqus import *
from abaqusConstants import *

# 创建模型
model = mdb.Model(name='CantileverBeam')

# 创建部件
part = model.Part(name='Beam', dimensionality=THREE_D, 
                  type=DEFORMABLE_BODY)
# 创建几何...
```

### 步骤三：发送提示词给AI

将以下内容发送给任意AI工具：

> 我需要写一个Abaqus脚本来做三维实体悬臂梁的静力分析。
> 请参考以下技能库模板来生成代码：
> 
> [复制 skill_modeling.md 中的建模代码]
> [复制 skill_material.md 中的材料代码]
> [复制 skill_static_linear.md 中的分析代码]
> 
> 要求：
> - 梁长度 500mm，截面 50x50mm
> - 材料：弹性模量 210GPa，泊松比 0.3
> - 左端固定，右端施加向下的力 1000N
> - 网格尺寸 10mm

### 步骤四：验证生成的代码

将AI生成的代码保存为 `.py` 文件，在 Abaqus CAE 中运行：
- `File` → `Run Script` → 选择生成的脚本
- 检查模型树、网格、结果是否正确

## 四、进阶技巧：提升使用效率的秘诀

### 技巧一：建立个人知识索引

不要把技能库当作一次性工具，而是建立长期的索引系统：

1. **按分析类型分类**：静力、动力、热力、疲劳等
2. **按功能模块分类**：建模、材料、网格、边界、后处理
3. **标记常用模板**：给高频使用的技能文件加书签

### 技巧二：自定义提示词模板

为常用场景创建固定的提示词模板：

```python
# 我的Abaqus脚本生成模板
TEMPLATE = """
我需要生成Abaqus Python脚本，请按以下格式生成：

## 分析类型
{analysis_type}

## 几何参数
{geometry_params}

## 材料参数
{material_params}

## 载荷条件
{load_params}

## 边界条件
{bc_params}

## 其他要求
{other_requirements}

请参考技能库中的以下文件：
{skill_files}
"""
```

### 技巧三：善用代码片段功能

将技能库中的常用代码片段保存到 IDE 的代码片段中：

- VS Code: `File` → `Preferences` → `User Snippets`
- PyCharm: `File` → `Settings` → `Editor` → `Live Templates`

这样就可以快速插入常用代码，不需要每次都问AI。

### 技巧四：构建本地问答系统

如果你有一定技术能力，可以搭建本地的问答系统：

1. 使用 LangChain + 本地 Embedding 模型
2. 导入所有技能文档
3. 部署为本地 Web 服务
4. 局域网内多设备访问

这样可以获得类似 Claude Code 的体验，同时完全掌控数据和隐私。
